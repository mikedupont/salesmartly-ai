# salesmartly-ai 项目计划

这份计划跟着当前代码进度走，目标是让后续开发有清晰优先级，不把功能堆成一团。

## 当前阶段

当前项目已经从“工程底座”推进到“可运营、可标注、可持续沉淀训练资产”的阶段。现在最重要的不是再堆功能，而是把现有链路做稳、做清、做可追溯。

已经完成的核心能力：

- SaleSmartly 消息接入
- 长期记忆主链路
- D1 结构化存储
- Cloudflare Vectorize 语义记忆
- Relationship State 关系状态机
- Dialogue Policy Engine 对话策略层
- Post-processing 回复收敛层
- Memory Writer 写回编排层
- `/admin` 记忆查看页
- `/admin/memory` 调试接口
- `/health` 健康检查
- 基础烟雾测试
- 增量摘要链路
- 训练样本采集与导出链路
- 偏好反馈写回链路
- 训练样本后台标注链路
- 身份追问安全分支与后处理拦截
- `status=unlabeled` 的未标注工作流
- 手动人物经历录入与查看入口
- FlirtFlip 风格种子生成与清洗链路
- 当前可直接训练的公开对话清洗版
- FlirtFlip 线上同步与 D1 落库链路
- EmpatheticDialogues 线上同步与 D1 落库链路
- 本地 `data/` 训练文件已取消，训练资产只保留在 D1 和导出流里
- 公开对话库已完成一次基础清理，FlirtFlip 保留 `9996` 条，EmpatheticDialogues 保留 `9605` 条

当前阶段的专业判断：

- 这不是 demo 了，已经是一个可持续运营的情感陪伴后端
- 主要风险不在“能不能跑”，而在“数据质量、记忆一致性、策略稳定性、标注效率”
- 接下来最值钱的工作是把训练闭环和后台运营闭环做扎实，而不是再继续堆表面能力

## 里程碑 1

目标：把“能看见、能排查、能定位”的基础补齐并稳定住。

已完成或正在完成：

- 记忆查看接口
- 记忆调试接口
- 调试页展示策略层结果
- 调试页展示事实、摘要、向量、关系状态
- 手动人物经历录入 / 查看
- 训练样本查看与导出
- 偏好反馈记录

下一步：

- 继续补全后台页信息密度
- 增加更直观的调试视图
- 补充训练数据质量统计

## 里程碑 2

目标：把向量记忆和事实记忆做得更稳定、更准。

重点方向：

- 检索排序优化
- 相似结果去重
- 召回阈值调优
- 不同记忆源权重区分
- 更好的 query probe 组合
- 人物经历与摘要的召回优先级整理

## 里程碑 3

目标：把后台做成真正可用的运营界面，而不只是调试页。

重点方向：

- 用户维度总览
- 关系状态展示
- 事实记忆编辑/查看
- 摘要历史查看
- 向量召回结果查看
- 单用户对话轨迹查看
- 人物经历的手工维护
- 训练样本的连续标注

## 里程碑 4

目标：把质量保障补起来，并形成日常可执行流程。

重点方向：

- 更完整的测试覆盖
- 冒烟测试扩展
- 监控快照增强
- 常见异常告警信息
- 线上问题定位路径
- 训练集清理规则固定化
- 公开数据与真实数据隔离规则明确化

## 后续增强方向

在基础稳定后，再继续做：

- 更自然的对话节奏
- 更细的陪伴风格控制
- 更稳定的长期关系演化
- 更稳定的身份追问处理，不在普通聊天中自曝系统实现
- 更好的 Worker 职责切分，把后台和训练路由继续下沉
- 更完整的多用户隔离
- 更适合商业化的权限和后台
- 人物经历编辑、删除、版本化
- 训练样本的批量复核和抽样审核

## 里程碑 5

目标：把模型本身也调成更像真人，而不是只靠 prompt 和后处理。

推荐路线：

- 收集真实偏好样本
  - 同一输入保留多条候选回复
  - 标出更自然、更少 AI 味的版本

- 轻量监督微调
  - 用 LoRA / PEFT 做小成本风格适配
  - 先把基础聊天语气调顺

- 偏好优化
  - 用 DPO 或 RLHF 继续把“像人”的回复往上推
  - 把客服味、模板味、追问过多压下去

- 推理期兜底
  - 继续保留 Dialogue Policy Engine
  - 继续保留 Post-processing
  - 继续保留身份追问专用安全分支
  - 让训练和策略双保险配合

这一步完成后，项目会从“规则和 prompt 驱动的陪聊系统”进化成“经过偏好训练的长期陪伴 AI”。

## 里程碑 6

目标：真正形成一个能不断长出训练资产的闭环。

重点方向：

- 自动导出训练数据集
- 偏好标签审核与回写
- 后台一键标注 chosen / rejected
- 只看未标注样本的连续标注模式
- 样本质量统计
- 好 / 坏回复对比分析
- 为 LoRA / DPO 准备干净数据
- 所有训练源都走线上同步，不再依赖本地数据目录
- 人物经历只进入长期记忆，不混入公开训练语料
- 训练实现细节见 [`TRAINING_IMPLEMENTATION.md`](TRAINING_IMPLEMENTATION.md)
- 当前公开对话清洗版已经生成，可直接拿去继续整理训练集

## 推荐开发顺序

1. 先稳住记忆查看、人物经历维护和调试接口
2. 再继续优化向量召回和事实记忆优先级
3. 再补后台批量操作和训练标注效率
4. 最后补测试、监控和告警

## 当前判断

这套系统已经不是“聊天机器人 demo”了，而是一个可运营、可持续标注、可持续训练的长期陪伴 AI 底座。
